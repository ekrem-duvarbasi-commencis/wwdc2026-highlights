Most Important

Siri becomes personal, contextual, and app-aware.

Siri now works with Apple Intelligence, App Intents, and app-provided schemas so it can understand user context and act across apps more naturally.

SiriApp Intents

Personal Intelligence across apps

Siri can use personal context from information like messages, email, calendar, and app entities to answer richer requests and complete longer tasks.

What you can do

  • Audit the moments where users currently copy information between your app, Mail, Calendar, Notes, or Messages.
  • Expose the content Siri needs as app entities instead of keeping it trapped in custom screens or local-only view state.
  • Design actions around user intent, such as create, summarize, schedule, book, draft, attach, or share.
Sources:Siri announcementApple Intelligence guide
SchemasSpotlight

App Schemas make app content actionable

Apps describe entities and actions through schemas so Siri, Shortcuts, and Spotlight can reason about real app content instead of fixed trigger phrases.

What you can do

  • Model your most important app objects as entities with stable IDs, display names, and query support.
  • Keep intents small and composable so Siri can chain them with system and third-party actions.
Swift shape 
struct OpenProjectIntent: AppIntent {
    static var title: LocalizedStringResource = "Open Project"

    @Parameter(title: "Project")
    var project: ProjectEntity

    func perform() async throws -> some IntentResult {
        try await ProjectNavigator.open(project.id)
        return .result()
    }
}

TestingReliability

App Intents can be validated through system paths

The AppIntentsTesting framework helps teams test Siri, Spotlight, and Shortcuts behavior without treating voice interactions as fragile UI automation.

What you can do

  • Add tests for entity lookup, missing-parameter prompts, authorization errors, and localized vocabulary.
  • Run these tests in CI before shipping new App Intent schemas or renaming user-visible actions.

On-device and private cloud AI become developer primitives.

WWDC26 adds deeper Foundation Models support, Core AI for custom models, evaluations for agentic behavior, and MLX workflows for Apple Silicon.

Foundation Models framework grows into agentic apps

Developers can build experiences that combine language models, app tools, structured responses, and evaluations while preserving Apple platform privacy expectations.

What you can do

  • Start with narrow, low-risk workflows where the model can draft, classify, summarize, or prepare actions for review.
  • Represent app capabilities as explicit tools so the model can request data or actions through controlled interfaces.
  • Return structured output when the result needs to drive UI, persistence, or follow-up automation.
Agent tool shape 
struct DraftReplyTool: AppAITool {
    let threadID: MessageThread.ID

    func call(context: ToolContext) async throws -> DraftReply {
        let thread = try await Messages.load(threadID)
        return try await ReplyComposer.draft(from: thread)
    }
}

Core AI runs custom models on Apple Silicon

Core AI is the lower-level path for loading, optimizing, and running custom models on-device when an app needs more control than prompt-based APIs provide.

What you can do

  • Plan model packaging, memory budgets, warm-up costs, and fallback behavior as part of the feature design.

Evaluations and MLX make AI behavior measurable

What you can do

Localization agents reduce string-catalog work

The App Store becomes more visual, measurable, and flexible.

App Store updates include richer product-page assets, Retention Messaging, more subscription patterns, new IAP capabilities, and expanded analytics.

Product pages and search get richer assets

Header images, search result videos, and centralized asset management help apps explain themselves visually before a user opens the full product page.

What you can do

  • Prepare product-page header images that show the actual app experience, not abstract brand art.
  • Plan short search videos around one clear conversion moment: create, scan, complete, compare, or share.
  • Use Asset Library to separate creative review from app binary releases where possible.
Asset checklist

Search video: 10-30 s, one feature, no intro
Custom page: audience-specific screenshots
Review: submit assets before campaign launch

Subscriptions support groups, organizations, and retention

Group and organization subscription options broaden who can buy access, while Retention Messaging helps developers communicate value at cancellation moments.

What you can do

  • Map whether your product is bought by individuals, families, teams, classrooms, or organizations.
  • Define entitlement rules for owner, member, invitee, expired seat, and transferred ownership states.
  • Write Retention Messaging around concrete value the user has already received, not generic discount pressure.

Safety, trust, and policy changes need early review.

WWDC26-era changes include Time Allowances, age assurance APIs, policy updates, trust tooling, and security guidance for agentic features.

Time Allowances classify app categories

iOS 27, iPadOS 27, and macOS 27 add Time Allowances so parents can manage time by categories like Entertainment, Games, and Social Media.

What you can do

  • Review App Store categories and age-rating answers before users see category-based time controls.
  • Audit whether feeds, profiles, reposting, comments, or messaging create social media capability obligations.
Sources:Apple Developer NewsApp Store guide

Age assurance APIs affect regulated regions

Apple points developers to Declared Age Range, PermissionKit significant-change APIs, age rating metadata, and server notifications for regional compliance.

What you can do

  • Request only the age category data your feature needs and keep region-specific logic centralized.
  • Use significant-change flows when a feature meaningfully changes what minors can access or do.
  • Handle server notifications so revoked consent updates your backend state quickly.

let ageRange = try await AgeRangeProvider.currentRange()

guard ageRange.allowsSocialFeatures else {

    return
}

Agentic features need security review

Apple includes guidance for securing agentic features, using App Attest, and understanding trust signals when apps rely on AI-driven actions.

What you can do

  • Separate suggested actions from committed actions when money, privacy, account state, or public content is involved.
  • Validate tool inputs server-side even when the request came from an on-device assistant.
  • Use App Attest and trust signals to reduce abuse of high-value endpoints.
