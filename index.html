WWDC26: All Systems Glow — iOS 27, Siri AI & macOS Golden Gate

Most Important

Siri becomes personal, contextual, and app-aware.

Siri now works with Apple Intelligence, App Intents, and app-provided schemas so it can understand user context and act across apps more naturally.

Personal Intelligence across apps

By adopting App Intents schemas, your app's content enters the Spotlight semantic index, enabling personal context understanding with attribution back to your app — and its actions become available through natural language without fixed trigger phrases.

  • Entity schemas contribute your app's content to the Spotlight semantic index so Siri can understand and surface it with attribution back to your app.
  • Intent schemas let people act on that content naturally — no specific phrases to define, and no code changes needed as Siri's language understanding evolves.
  • The View Annotations API adds on-screen awareness: map your views to entities so people can reference and act on whatever is visible on screen conversationally.
Sources:Siri announcementApple Intelligence guide

App Schemas make app content actionable

Apps describe entities and actions through schemas so Siri, Shortcuts, and Spotlight can reason about real app content instead of fixed trigger phrases.

  • Important app objects need stable identities and readable names so Siri and Spotlight can find the same things users see in the interface.
  • Smaller, composable intents work better because the system can combine them with other app and system actions.
Sources:Build Siri experiencesAdvanced App Intents

App Intents can be validated through system paths

The AppIntentsTesting framework helps teams test Siri, Spotlight, and Shortcuts behavior without treating voice interactions as fragile UI automation.

  • Entity lookup, missing information, permissions, and localized vocabulary are the places Siri integrations tend to feel brittle.
  • Testing those paths makes App Intents feel more like normal app code and less like a voice-only mystery box.
Sources:AppIntentsTestingCode-along: Siri

On-device and private cloud AI become developer primitives.

WWDC26 adds deeper Foundation Models support, Core AI for custom models, evaluations for agentic behavior, and MLX workflows for Apple Silicon.

Foundation Models framework grows into agentic apps

Developers can build experiences with Apple Foundation Models, other language model providers, app tools, multimodal prompts, Dynamic Profiles, and evaluations.

  • Multimodal prompts let apps reason over images and text together, while Vision tools such as OCR and barcode reading can be available on-device.
  • Dynamic Profiles let an app change models, tools, and instructions during a continuous session.
  • The Evaluations framework gives teams a source-backed way to test AI behavior beyond normal unit tests.

Core AI runs custom models on Apple Silicon

Core AI is a new OS-level framework for bringing your own models on-device with a memory-safe Swift API built for Apple Silicon.

  • Apple calls out zero server dependencies, zero token costs, automatic hardware specialization, and ahead-of-time compilation for quick load times.
  • Fine-grained memory control, zero-copy data paths, and stateful execution make it relevant for both compact models and larger generative workloads.
Sources:Meet Core AIOn-device AI modelsModel optimization

Evaluations and MLX make AI behavior measurable

Evaluations help validate prompts and intelligence-powered features before they reach users, while MLX expands local and distributed model experimentation on Apple Silicon.

  • Evaluations are meant to make prompt and agent behavior testable inside the development workflow.
  • MLX adds support for Metal 4, GPU Neural Accelerators, and distributed training across multiple Macs using RDMA over Thunderbolt.

Xcode 27 turns agentic coding into a daily workflow.

Agents help with prototypes, repetitive code, localization, testing, and diagnostics while Device Hub centralizes simulator and physical-device work.

Coding agents in Xcode

Xcode agents can help generate UI prototypes, explain project context, perform repetitive implementation work, and assist across the development cycle.

  • Agents shine on focused work: repetitive migrations, prototype views, test scaffolding, and documentation updates.
  • Clear context helps a lot. A good request names the files, the expected behavior, and the tests that should pass.
  • The developer still owns the result: diffs, tests, and architecture decisions remain part of normal review.
Sources:What is new in Xcode 27Xcode, agents, and youUI prototypes using agents

Localization agents reduce string-catalog work

Localization agents can translate strings, add languages, and handle plural variants with app context and Apple style guidance.

  • Localization works better when strings already live in String Catalogs instead of being scattered through views.
  • Context matters for words like "Archive", "Draft", or "Charge", where the right translation depends on the screen.
  • Plural rules, purchase copy, privacy text, and legal language still deserve a careful human pass.
Sources:Translate using agentsXcode guide

Device Hub speeds up device and simulator workflows

Device Hub is Xcode's central workspace for managing development devices and simulators, making it easier to prepare test targets, inspect their state, and move quickly from setup to bug reproduction.

  • Developers can control connected devices and simulators from one place instead of jumping between separate Xcode windows and system tools.
  • Organization and configuration tools help keep the right devices, runtimes, and testing setups ready for repeated debugging work.
  • Device Hub pairs with devicectl when teams need command-line device management or automation around the same workflows.
Sources:Device HubDevice Hub docs

Customizable toolbar keeps Xcode workflows closer

Xcode 27 revamps the workspace toolbar so core navigation, editor controls, coding-agent access, and run destination choices are easier to reach and arrange around the way you work.

  • History navigation and editor controls move from the jump bar into the toolbar, while build activity appears beneath the window title.
  • The center area includes the coding-agent entry point plus the scheme and destination picker.
  • Top-right controls add tabs and editor panes, customize editor settings, and switch between canvas, Assistant Editor, and source-control review modes.
  • The branch picker moves to the bottom bar, and toolbar items can be added, removed, and reordered.

Safety, trust, and policy changes need early review.

WWDC26-era changes include Time Allowances, age assurance APIs, policy updates, trust tooling, and security guidance for agentic features.

Time Allowances classify app categories

iOS 27, iPadOS 27, and macOS 27 add Time Allowances so parents can manage time by categories like Entertainment, Games, and Social Media.

  • App Store categories and age-rating answers become more visible when parents manage time by app category.
  • Feeds, profiles, reposting, comments, and messaging can affect whether an app is treated as having social media capabilities.
Sources:Apple Developer NewsApp Store guide

Age assurance APIs affect regulated regions

Apple points developers to Declared Age Range, significant update flows, PermissionKit resources, age rating metadata, and sandbox testing for regional requirements.

  • The Declared Age Range API requests an age range while preserving privacy and requires the declared-age-range entitlement.
  • Apple documents Significant Update Action for significant app update acknowledgments.
  • Apple’s age assurance Q&A directs technical implementation questions to Declared Age Range and PermissionKit resources.
Sources:Apple Developer NewsAge assurance Q&ADeclared Age Range

Agentic features need security review

Apple includes guidance for securing agentic features, using App Attest, and understanding trust signals when apps rely on AI-driven actions.

  • The session focuses on indirect prompt injection risks such as data exfiltration and unintended actions.
  • Apple highlights threat modeling, secure prompt design, authentication, and user confirmations as mitigation themes.
  • The guidance specifically connects these risks to App Intents and the Foundation Models framework.
